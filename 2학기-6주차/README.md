# HTML 메일 전송 프로그램 (문제2: 감동의 메시지)

## 개요
화성에서 온 Dr. Han 박사의 감동적인 메시지를 HTML 형식으로 여러 수신자에게 전송하는 프로그램입니다.

## 주요 기능
- HTML 형식의 아름다운 메일 전송
- CSV 파일에서 수신자 목록 읽기
- 두 가지 메일 전송 방법 제공
- PEP 8 스타일 가이드 준수

## 파일 구조
```
2학기-6주차/
├── sendmail_html.py      # 메인 프로그램
├── mail_target_list.csv  # 수신자 목록 CSV 파일
└── README.md            # 이 파일
```

## 설치 및 설정

### 1. Gmail 앱 비밀번호 설정
1. Gmail 계정에 로그인
2. 계정 설정 → 보안 → 2단계 인증 활성화
3. 앱 비밀번호 생성 (메일용)
4. 생성된 16자리 앱 비밀번호 저장

### 2. CSV 파일 준비
`mail_target_list.csv` 파일을 다음과 같은 형식으로 준비:
```csv
이름,이메일
김철수,kim@example.com
이영희,lee@example.com
박민수,park@example.com
```

## 사용 방법

### 1. 프로그램 실행
```bash
python sendmail_html.py
```

### 2. 계정 정보 입력
- 보내는 사람 이메일 주소: Gmail 주소 입력
- 이메일 앱 비밀번호: 위에서 생성한 앱 비밀번호 입력
- 메일 제목: 원하는 제목 입력 (기본값: "화성에서 온 메시지")

### 3. 전송 방법 선택
1. **방법 1**: 받는 사람에 여러명을 열거하는 방법
2. **방법 2**: 한번에 한 명씩 메일을 반복적으로 보내는 방법
3. **방법 3**: 두 방법 모두 시도하여 비교

## 메일 전송 방법 비교

### 방법 1: 받는 사람에 여러명을 열거
- **장점**: SMTP 연결을 한 번만 설정하여 효율적
- **단점**: 한 명의 이메일 주소가 잘못되면 전체 전송 실패 가능
- **적합한 경우**: 수신자 목록이 신뢰할 수 있고 안정적인 경우

### 방법 2: 한번에 한 명씩 반복 전송
- **장점**: 각 수신자마다 독립적으로 전송되어 안정적
- **단점**: SMTP 연결을 여러 번 설정하여 상대적으로 비효율적
- **적합한 경우**: 수신자 목록에 오류가 있을 수 있는 경우

## 테스트 방법

### 1. 기본 테스트
```bash
# 프로그램 실행
python sendmail_html.py

# 테스트용 계정 정보 입력
보내는 사람 이메일 주소: your_email@gmail.com
이메일 앱 비밀번호: your_app_password
메일 제목: 테스트 메시지

# 전송 방법 선택 (3번 추천)
선택 (1/2/3): 3
```

### 2. CSV 파일 테스트
1. `mail_target_list.csv` 파일에 실제 이메일 주소 입력
2. 프로그램 실행하여 실제 전송 테스트
3. 수신자들이 HTML 메일을 정상적으로 받았는지 확인

### 3. 오류 처리 테스트
1. 잘못된 이메일 주소를 CSV에 포함
2. 잘못된 앱 비밀번호 입력
3. 네트워크 연결 끊김 상황 시뮬레이션

## 문제 해결

### 일반적인 오류
1. **인증 오류**: 앱 비밀번호가 올바른지 확인
2. **CSV 파일 오류**: 파일 형식과 인코딩 확인
3. **네트워크 오류**: 인터넷 연결 상태 확인

### 디버깅 팁
- 테스트용 이메일 주소로 먼저 시도
- CSV 파일의 첫 번째 행이 헤더인지 확인
- 이메일 주소 형식이 올바른지 확인
